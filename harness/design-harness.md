## Brand Tone

사용자가 원하는 브랜드 톤은 이후 확정하되, 초기 방향은 아래 기준으로 정한다.

- 과장된 수익 약속보다 신뢰, 검증, 투명성을 우선한다.
- AI 모델 마켓플레이스처럼 탐색이 쉬워야 하지만, 투자 앱답게 위험 고지를 명확히 보여준다.
- 전문적이되 차갑지 않고, 초보자도 정보 구조를 따라갈 수 있게 한다.

## Target Impression

앱을 처음 접했을 때 사용자가 느끼길 원하는 첫인상은 다음과 같다.

- “AI 모델을 골라 투자 전략을 위임하는 앱”이라는 핵심이 첫 화면에서 드러난다.
- 돈, 모델 성과, 위험 등급, 운용 범위가 빠르게 비교된다.
- 게임처럼 보이지 않고, 금융 서비스의 조심스러운 밀도를 유지한다.

## Observed Layout Patterns

design-samples/selected/의 모바일 캡처 20장을 기준으로 관찰한 패턴은 다음과 같다.

- 1170 x 2532 비율의 세로형 모바일 화면이며, 앱 UI 기준으로는 약 390 x 844 프레임에 대응한다.
- 화면 상단에는 큰 페이지 제목과 우측 아이콘 버튼이 반복되고, 하단에는 4~5개의 탭 내비게이션이 고정된다.
- 배경은 대부분 흰색 또는 아주 옅은 회색/블루 톤이고, 핵심 정보는 둥근 카드와 리스트 행으로 묶어 보여준다.
- 카드에는 넓은 여백, 20px 전후의 큰 라운드, 미묘한 그림자 또는 구분선이 사용된다.
- 실시간 이슈/랭킹 화면은 상단 컬러 영역과 흰색 카드 리스트를 결합한다.
- AI 소개/추천 화면은 연한 블루 톤 배경 위에 큰 헤드라인과 세로형 기능 카드로 구성된다.
- 피드 화면은 게시물 카드, 태그, 미디어 썸네일, 반응 버튼을 포함한다.
- 계좌/홈 화면은 시장 지수, 알림 배너, 계좌 카드, 자산 리스트를 한 화면에 압축한다.
- 리스트 행은 좌측 순위/아이콘, 중앙 제목/설명, 우측 수치/상태값의 3단 구조가 많다.

## Initial Screen Structure

초기 디자인은 다음 화면 구조로 가져간다.

1. Home / My AI Investment: 선택된 AI 모델, 예치금, 오늘의 모델 신호, 최근 운용 상태를 보여준다.
2. Discover Models: 사람들이 올린 AI 모델을 위험도, 시장, 전략 태그, 누적 성과 기준으로 탐색한다.
3. Realtime Signals: 뉴스 트래픽, 가격 추세, 시장 이슈를 모델 관점으로 랭킹화한다.
4. Model Detail: 모델 설명, 투자 대상, 레버리지 여부, 고위험 여부, 성과, 위험 고지를 명확히 보여준다.
5. Feed / Insights: 모델 운용 코멘트와 시장 뉴스 요약을 피드 형태로 제공한다.

## Color Palette

관찰 기반 후보 색상이며, 최종 브랜드 컬러는 사용자가 확정해야 한다.

- 배경: #FFFFFF, #F7F9FC, #F2F6FF
- 표면: #FFFFFF, #F4F6FA
- 구분선: #E9EEF5
- 주요 액션/강조: #2F80ED, #4C8DFF
- 상승/위험 강조: #F04452
- 하락/보조 강조: #3182F6
- 본문 텍스트: #202632
- 보조 텍스트: #7D8796
- 비활성 텍스트: #A5ADBA

## Typography

폰트는 아직 확정하지 않는다. 초기 Figma 구조에서는 Inter를 사용하고, 실제 서비스에서는 한국어 가독성이 좋은 폰트로 교체한다.

- 화면 타이틀: 30~36px, Bold
- 섹션 제목: 20~24px, Bold 또는 Semi Bold
- 카드 제목: 16~18px, Semi Bold
- 본문/리스트: 14~16px, Regular 또는 Semi Bold
- 보조 설명/배지: 11~13px, Medium
- 숫자/수익률: 굵기와 색으로 빠르게 구분하되 과도한 장식은 피한다.

## Components

초기 예상 컴포넌트는 다음과 같다.

- MobileShell: 모바일 화면 프레임, 안전 영역, 하단 탭 포함
- TopIconBar: 검색, 알림, 설정 등 상단 아이콘 영역
- BottomNav: 홈, 모델, 신호, 피드, 내 투자 탭
- SoftBanner: AI 시그널, 공지, 리스크 안내 배너
- MetricCard: 예치금, 누적 수익률, 변동성, 모델 운용 상태 카드
- ModelCard: AI 모델명, 운용 범위, 위험도, 누적 성과, 설명 요약
- RiskBadge: 레버리지, 미국장, 채권형, 고위험 등 모델 전략 태그
- SignalRankCard: 뉴스/가격/트래픽 기반 이슈 랭킹 카드
- FeedPost: 모델 운용 코멘트, 뉴스 요약, 커뮤니티형 피드 카드
- ModelDetailMandate: 모델이 정한 투자 원칙, 허용 자산, 제한 사항 설명 블록
- DisclosurePanel: 투자 위험, 성과 비보장, 모델 책임 범위 안내
- SectionHeader: 섹션 제목과 설명을 정리하는 헤더
- ChipTab: 위험도, 시장, 전략 등 필터링 탭
- FloatingActionButton: 핵심 액션 진입 버튼

## UI/UX Improvement Loop Criteria

5분 단위 자동 UI/UX 개선 루프는 아래 기준을 따른다.

- 한 번의 루프에서는 Home, Discover Models, Model Detail, Portfolio, Signals, Feed 중 한 화면 또는 한 공통 컴포넌트만 다룬다.
- 참고 캡처에서는 밝은 배경, 선명한 숫자 계층, 절제된 카드 구분, 하단 탭, 좌측 제목과 우측 수치 정렬 같은 관찰 가능한 금융앱 패턴만 참고한다.
- 토스 증권의 로고, 고유 문구, 정확한 화면 구성, 브랜드 컬러 조합, 아이콘 배치를 그대로 복제하지 않는다.
- mock, sample, prototype 설명은 안전 경계가 필요한 위치에만 보조 라벨이나 상세 안내로 낮춰 배치한다.
- 실입금, 실주문, 브로커 연결, 투자 권유처럼 오해될 수 있는 버튼이나 문구는 만들지 않는다.
- 주요 카드와 리스트 행은 390px 모바일에서 제목, 보조 설명, 수치, 상태가 겹치지 않도록 3단 정보 구조를 우선한다.
- 클릭 가능한 카드, 필터, 하단 탭은 150~220ms 수준의 hover, pressed, focus 피드백을 사용하고 reduced-motion 대응을 유지한다.
- 리스트와 피드는 추천이나 매매 지시보다 관찰, 상태, 맥락, 모델 운용 범위를 읽히게 정리한다.
- 작업 후에는 타입 검사와 390px visual structure smoke 기준을 통과해야 하며, 실패하면 다음 루프에서 작은 수정 항목으로 나눈다.

## Open Placeholders

다음 항목은 사용자가 확정해야 한다.

- 브랜드 로고와 최종 앱 아이콘
- 최종 브랜드 컬러
- 한국어 서비스 카피 톤
- 실제 투자 위험 고지 문구
- 모델 성과 산식과 표기 기준
- 앱 첫 출시 범위가 웹, 모바일 웹, 네이티브 앱 중 무엇인지

## Do Not

- 수익을 보장하는 듯한 시각 표현이나 문구를 사용하지 않는다.
- 위험 정보, 수수료, 모델 운용 범위, 투자 제한 사항을 작게 숨기지 않는다.
- 레버리지/고위험 모델을 게임처럼 보이게 만들지 않는다.
- 참고 스크린샷의 고유 브랜드, 로고, 문구, 구체적인 화면 구성을 그대로 복제하지 않는다.
- 사용자가 정해야 하는 브랜드 철학, 카피, 로고, 최종 컬러를 임의로 확정하지 않는다.
