## Required Caution

- 국가, 투자상품 유형, 자산군, 주문 실행 방식에 따라 인허가/등록/공시/적합성 의무가 달라질 수 있습니다.
- Codex는 법률 판단을 확정하지 않습니다.
- 규제와 직접 연결되는 기능은 `legal_review` 또는 `financial_operation` 상태로 기록합니다.
- MVP 경계와 금지 기능은 `harness/domain-contract-harness.md`를 따릅니다.

## Sensitive Features

- 사용자 자금 입금/출금
- 실제 매매 주문
- 브로커 또는 은행 계좌 연결
- 수익 보장처럼 보이는 광고 문구
- 모델 성과 claim
- 추천/권유로 해석될 수 있는 문구
- 고위험/레버리지 모델 노출
- 외부 AI 모델 파일 실행

## Required Product Safeguards

- 모델 설명 페이지에는 위험, 투자 범위, 금지 자산, 레버리지 여부, 손실 가능성을 표시합니다.
- 모델의 과거 성과는 미래 수익 가능성과 혼동되지 않게 표시합니다.
- 사용자가 모델의 전략을 직접 바꾸는 UI를 제공하지 않습니다.
- 고위험 모델은 추가 고지와 심사 상태를 요구합니다.
- `TradeIntent`는 실제 주문이 아니라 주문 전 의사결정/시뮬레이션 단계로만 표현합니다.
